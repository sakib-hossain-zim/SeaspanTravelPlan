General Information - top

Before starting to work with template, there are some basic concepts/technologies you need to be familiar with. Please check the Susy framework and SASS for more information. These two technologies and the core of the template.

There are some other essentials technologies used across the template files. Material Icons are used for displaying the icons. However these technogies are not mandatory to use. But learning these technologies will make easier site creation.

Color Combinations - top

CSS folder contains multiple color combinations so you can use already prepared colors. Everything what you need to do is to rename particles.css to match filename.

By using the gulpfile.js in "react" folder you can create or modify already existing sass files. The CSS files inside "html/assets/css" directory are just copied from the "react/public/css" folder.

Optional Classes - top

You can set toolbar actions to slide from the left side instead of poping out by using CSS class "toolbar-slide" on ".toolbar" div.

File Structure - top

All CSS files are located inside assets/css folder. Template oriented files are generated from SASS source codes. You can find the SASS files insidede public/scss. It is recommended to edit SASS files instead of CSS files because if you compile the SASS, you will loose all your changes.

Images are located in public/img. The tmp folder container sample images. In the package which you downloaded are situated the compressed images with the placeholders only.

Folder Description
public/css CSS files
public/data JSON data for eg. contacts
public/img Images
public/scss SASS files

Application Installation - top

Installing on the local machine

Please note that we will be working in terminal. Mac or Linux users have already terminal available in OS but Windows users must look for third party solution

  1. cd particles
  2. npm install # Install all dependencies
  3. npm run
  4. npm run-script build # Create production build

Now you can access http://localhost:3000

Installing on the hosting

When installing application on the hosting you can replicate almost all steps from the previous guide how to install it on the local machine but the last command (server:run) is not recommended to run. You must configure your own webserver. For example our demo is running on nginx. Below we attached our nginx configuration.

Nginx Configuration - top

server {
    listen 80;
    listen [::]:80;

    root /PATH_HERE/particles/build;
    index index.php;
    client_max_body_size 20M;

    gzip                on;
    gzip_min_length     1000;
    gzip_buffers        4 8k;
    gzip_http_version   1.0;
    gzip_disable        "msie6";
    gzip_types          text/plain text/css application/json application/javascript application/x-javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_vary           on;

    server_name YOUR_DOMAIN_HERE;

    location / {
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
